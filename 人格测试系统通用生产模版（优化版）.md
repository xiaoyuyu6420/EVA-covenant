## 二、设计维度体系

### 2\.1 选定模型数量

```Plain
模型数量 = [MODEL_COUNT]   // 建议 4~6 个
```

**每个模型代表一个行为大类**，彼此正交（尽量不重叠）。常见模式：

|模式|示例|
|---|---|
|游戏|沉迷 / 社交 / 策略 / 情感 / 风格|
|职场|执行 / 沟通 / 决策 / 学习 / 价值观|
|消费|冲动 / 比价 / 品牌 / 体验 / 社交|
|社交|主动 / 共情 / 边界 / 表达 / 忠诚|

### 2\.2 每模型拆维度

```Plain
每模型维度数 = [DIM_PER_MODEL]   // 建议 3 个
总维度数 = MODEL_COUNT × DIM_PER_MODEL
```

每个维度是模型下可量化的一个方面：

```Plain
模型 [M1_NAME]:
  [M1_D1]  [M1_D1_DESC]
  [M1_D2]  [M1_D2_DESC]
  [M1_D3]  [M1_D3_DESC]

模型 [M2_NAME]:
  [M2_D1]  [M2_D1_DESC]
  [M2_D2]  [M2_D2_DESC]
  [M2_D3]  [M2_D3_DESC]

... 重复至 MODEL_COUNT 个模型
```

**维度命名规则：** `\[模型字母\]\[序号\]`，如 A1, A2, A3, S1, S2, S3\.\.\.

### 2\.3 定义维度权重

```Plain
全部维度默认权重 1.0
对分组区分起决定作用的维度权重 ↑ (1.5~2.0)
对分组区分无关的维度权重 ↓ (0.5~0.8)

权重数组: [WEIGHTS]   // 长度 = 总维度数
```

> 判断方法：问自己 \&\#34;区分 A 组和 B 组，看哪个维度最关键？\&\#34; 那个维度权重拉高。

## 三、设计人格类型

### 3\.1 确定类型数量

```Plain
常规人格数 = [TYPE_COUNT]       // 建议 20~30 种
特殊人格数 = [SPECIAL_COUNT]    // 建议 1~3 种
分组数 = [GROUP_COUNT]          // 通常 = 模型数 + 1 (兜底)
```

**分组规则：** 每组对应 \&\#34;某个模型维度最突出\&\#34; 的人格集合，加一个 \&\#34;无突出\&\#34; 的兜底组。

### 3\.2 填充人格模板

每个类型需定义：

```Plain
代码:    [CODE]        // 4 字母缩写，唯一
名称:    [NAME]        // 2~4 字，有记忆点
分组:    [GROUP]       // 所属分组
向量:    [VECTOR]      // 格式见下方
标语:    [SLOGAN]      // 一句话人设
描述:    [DESC]        // 2~3 句展开描述
```

**向量写法：** 按模型分组，每维度一个字母 \(L/M/H/X\)，用 `\-` 分隔：

```Plain
[MMM]-[MMM]-[MMM]-[MMM]-[MMM]
  │      │      │      │      └─ 模型5: [M5_D1][M5_D2][M5_D3]
  │      │      │      └──────── 模型4: [M4_D1][M4_D2][M4_D3]
  │      │      └─────────────── 模型3: [M3_D1][M3_D2][M3_D3]
  │      └────────────────────── 模型2: [M2_D1][M2_D2][M2_D3]
  └───────────────────────────── 模型1: [M1_D1][M1_D2][M1_D3]
```

**向量设计原则：**

- 每组人格在该组对应的模型维度上拉高 \(H/X\)，其他模型维度中等或偏低

- 同组内的人格通过 \&\#34;次要模型维度的差异\&\#34; 来区分

- 确保任意两个人格的向量至少有 3 个维度不同（否则合并）

### 3\.3 设计特殊人格

每个特殊人格需定义：

```Plain
代码:    [SPECIAL_CODE]
名称:    [SPECIAL_NAME]
触发方式: [TRIGGER_TYPE]   // "gate+trigger" 或 "fallback"
触发条件: [TRIGGER_COND]   // 如 "门控题选X + 触发题选Y" 或 "最佳匹配 < 阈值"
```

## 四、设计题目

### 4\.1 常规题

```Plain
每维度题数 = [Q_PER_DIM]      // 建议 3 题
每题选项数 = [OPTIONS_PER_Q]  // 建议 3 个
每选项分值 = 1 / 2 / 3

总常规题数 = 总维度数 × Q_PER_DIM
```

每道题的格式：

```Plain
{
  dim: "[DIM_CODE]",           // 如 "A1"
  text: "[QUESTION_TEXT]",     // 有代入感的场景题
  options: [
    { label: "[OPT_A]", score: [1|2|3] },  // 低分选项
    { label: "[OPT_B]", score: [1|2|3] },  // 中分选项
    { label: "[OPT_C]", score: [1|2|3] },  // 高分选项
  ]
}
```

**出题原则：**

- 用场景而非直接提问（\&\#34;你打到最后关头发现策略错了，你会？\&\#34; 而非 \&\#34;你灵活吗？\&\#34;）

- 三个选项分别映射低 / 中 / 高，语言风格统一

- 同维度两题从不同角度测（避免用户猜到意图后迎合）

- 第 3 题做反向校验（与前面两题的措辞方向相反）

### 4\.2 门控题

```Plain
{
  text: "[GATE_QUESTION]",      // 如 "你最喜欢做什么？"
  options: [
    { label: "[OPT_1]", value: "[PATH_1]" },  // 触发路径 1
    { label: "[OPT_2]", value: "[PATH_2]" },  // 触发路径 2
    { label: "[OPT_3]", value: "[PATH_3]" },  // 触发路径 3
    { label: "[OPT_4]", value: "normal" }      // 不触发，正常走向量匹配
  ]
}
```

- 插入位置：总题数约 2/3 处 ± 随机偏移 2\~3 题

- 至少 1 个选项触发特殊路径，其余走正常匹配

- 未触发路径的选项值可做匹配微调信号

### 4\.3 触发题

每个可触发的路径配一道触发题：

```Plain
{
  text: "[TRIGGER_QUESTION]",
  options: [
    { label: "[YES_OPT]", trigger: "[SPECIAL_CODE]" },  // 触发特殊人格
    { label: "[NO_OPT]" }                                 // 不触发
  ]
}
```

## 五、算法配置

### 5\.1 分数 → 向量转换

```Plain
每维度总分 = Q_PER_DIM × 最高分
总分范围 = Q_PER_DIM ~ (Q_PER_DIM × 3)

分档规则 (以 3 题 / 总分 3~9 为例):
  3~4 → L (0)
  5~6 → M (1)
  7~8 → H (2)
  9   → X (3)

通用公式:
  档位数 = 选项数 (通常 4 档)
  档位边界 = 总分范围按档位数均分
```

### 5\.2 距离计算

```Plain
加权曼哈顿距离:
  D = Σᵢ wᵢ × |uᵢ - tᵢ|     (i = 0..维度数-1)

  wᵢ = 第 i 维的权重 (默认 1.0)
  uᵢ = 用户第 i 维的档位值
  tᵢ = 模板第 i 维的档位值

  maxDist = Σᵢ wᵢ × (档位数 - 1)

相似度:
  similarity = (1 - D / maxDist) × 100%
```

### 5\.3 结果判定

```Plain
判定优先级:
  ① 特殊触发 → 命中任一触发标记 → 返回特殊人格, 相似度 100%
  ② 向量匹配 → 计算所有模板的距离, 取 Top 3
  ③ 边界检查 → Top1 与 Top2 相似度差 < Δ?
     YES + Top1 < T → 兜底人格
     YES + Top1 ≥ T → Top1, 标记"边界型"
     NO → 直接返回 Top1
  ④ 附加输出 → Top 3 + 群体定位 (第 N 个 / 占比 X%)

需要校准的参数:
  Δ = [DELTA]    // Top1/Top2 差距阈值, 建议 5~10%
  T = [THRESHOLD] // 兜底相似度下限, 建议 50~65%
```

## 八、运营与数据增长（新增：提升回头率 / 传播 / 数据透明）

### 8\.1 用户留存（回头率）设计

针对测试类产品 “一次性消费” 的痛点，通过以下机制提升用户复访与复测率：

```Plain
留存机制配置:
  历史存档: [HISTORY_ENABLED]       // 是否开启用户历史结果存档 (默认开启)
  复测支持: [RETEST_ENABLED]        // 是否支持复测，展示结果变化 (默认开启)
  内容推荐: [RECOMMEND_ENABLED]     // 是否开启同IP/同主题测试推荐 (默认开启)
  更新提醒: [NOTIFY_ENABLED]        // 是否开启新测试上线提醒 (可选)
  社交对比: [MATCH_ENABLED]         // 是否开启好友结果匹配对比 (可选)
```

**核心策略：**

1. **结果沉淀**：用户可在个人中心查看所有历史测试结果，支持跨测试结果对比，将单次测试结果转化为长期个人档案

2. **复测激励**：支持用户间隔 7/30 天后复测，自动生成「变化报告」，展示人格维度的变化，激发用户 “看看自己有没有变” 的好奇心

3. **内容闭环**：完成当前测试后，推荐 2\~3 个同主题 / 同 IP 的关联测试，例如做完「明日方舟人格测试」后推荐「原神角色适配测试」，引导用户持续参与

4. **社交钩子**：支持用户邀请好友测试，生成「好友匹配度报告」，用户会主动回访查看好友的测试结果，大幅提升 7 日留存

### 8\.2 传播分析体系

构建全链路传播追踪体系，量化传播效果，优化裂变效率：

```Plain
传播配置:
  链路追踪: [TRACK_SHARE]          // 是否开启分享链路追踪 (默认开启)
  事件埋点: [TRACK_EVENTS]         // 是否开启行为事件埋点 (默认开启)
  渠道统计: [TRACK_CHANNEL]        // 是否开启来源渠道统计 (默认开启)
```

**核心能力：**

1. **裂变链路追踪**：

    - 每个分享链接自动携带追踪参数：`utm\_source=share\&amp;utm\_medium=\[渠道\]\&amp;share\_by=\[匿名分享者ID\]`

    - 支持追踪裂变层级：例如 A 分享给 B，B 分享给 C，完整记录传播链路，计算每个用户的传播影响力

2. **传播事件埋点**：

    - 统计核心事件：分享按钮点击、分享成功、结果截图、好友邀请点击

    - 按渠道拆分数据：微信 / 微博 / QQ / 朋友圈等，分析不同渠道的传播效率

3. **传播漏斗分析**：

    - 完整追踪转化漏斗：`曝光 → 进入测试 → 开始答题 → 完成答题 → 点击分享 → 分享成功 → 新用户通过分享进入`

    - 定位流失环节，例如如果 “完成答题→点击分享” 转化率低，可优化分享素材

### 8\.3 数据透明化设计

通过公开匿名统计数据，提升用户信任感，同时强化测试的社交热度感知：

```Plain
透明化配置:
  全局数据展示: [GLOBAL_STATS]     // 是否展示全局参与数据 (默认开启)
  人格分布展示: [TYPE_DIST]        // 是否展示人格类型占比分布 (默认开启)
  维度对比展示: [DIM_COMPARE]      // 是否展示用户与群体的维度对比 (默认开启)
  隐私说明: [PRIVACY_NOTICE]       // 是否展示隐私合规说明 (默认开启)
```

**展示内容：**

1. **热度数据**：在结果页顶部展示：`已有 \[TOTAL\] 人参与测试，今日新增 \[TODAY\] 人`，让用户直观感知测试的火爆程度

2. **人格分布**：用交互式饼图 / 柱状图展示所有人格类型的占比，例如：

    > 你的「\[人格名称\]」是第 3 热门的人格，占所有参与者的 12\.5%，全球只有 1/8 的人和你一样
    > 
    > 

3. **维度对比**：在维度雷达图中，叠加群体平均得分线，让用户直观看到自己的维度得分在人群中的位置，例如：

    > 你的「社交性」得分高于 85% 的参与者
    > 
    > 

4. **隐私保障**：同时展示隐私说明：`所有统计数据均为匿名聚合，不会泄露您的个人信息，符合GDPR/个人信息保护法要求`，消除用户的隐私顾虑

### 8\.4 访问量与行为统计

全面监控网站访问与用户行为，为产品优化提供数据支撑：

```Plain
统计指标:
  基础访问: PV/UV/平均停留时长
  答题行为: 答题完成率/单题放弃率/平均答题时长
  来源渠道: 直接访问/搜索引擎/社交平台/分享链接
  设备适配: 移动端/PC端占比/浏览器兼容性
```

**核心价值：**

- 通过访问量数据，监控测试的传播热度，及时调整运营策略

- 通过答题行为数据，优化题目设计：如果某道题的放弃率超过 20%，说明题目可能存在歧义或过长，需要调整

- 通过渠道数据，聚焦高转化的流量渠道，加大运营投入
